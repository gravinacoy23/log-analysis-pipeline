# Metric analysis

**Response time**

After analyzing the response time plot generated by the pipeline I noticed that most
of the logs are in the high end of the response time. Note that this is expected since
the response time is generated by either the CPU or the Memory being high, since these
are randomly generated numbers the probability of CPU or MEM being high is greater that
only CPU, for example, as the original design.

**CPU**

Originally I analyzed this metric with a very small set of logs (Aprox. 600), this generated
a weird plot of the RT distribution. However after generating a large enough dataset
(Aprox. 16000) I noticed that the CPU is evenly distributes. This is consistent with the nature
of the log generator, generates a random number for CPU. Values are between 30 and 70.

**Memory**
Findings are the same as with CPU, these two metrics are generated in the same way. The only
difference is that Memory has a range of 40, 75.
