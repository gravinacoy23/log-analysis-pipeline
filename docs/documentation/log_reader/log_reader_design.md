# Log Reader — Implementation (v1)

## Objective

Implement a log reader module for the airline booking backend pipeline.

The goal of this phase is to:

- Read synthetic log files generated by `log_generator.py`
- Return raw log lines as a list of strings
- Handle missing or empty log directories gracefully
- Prepare data for the parsing stage

---

# System Context

The log reader is the first stage of the ingestion layer.

It sits between the raw log files on disk and the parser. Its only
responsibility is to access the file system and deliver raw content.
It does not know anything about the log format or structure — that is
the parser's responsibility.

```
log_generator.py → data/raw/<service>/ → log_reader.py → log_parser.py
```

# Interface

## `load_service_logs(service)`

Reads the first available log file for a given service.

**Parameters:**
- `service` (str) — name of the service to read logs from (e.g. `"booking"`)

**Returns:**
- List of strings — one string per log line

**Raises:**
- `ValueError` — if the service directory does not exist
- `FileNotFoundError` — if the service directory exists but contains no log files

# Implementation Details

## Path Resolution

Uses `pathlib` for OS-independent path resolution. Resolves the project
root dynamically using `__file__` to ensure portability across
environments including Docker.

```
src/ingestion/log_reader.py
    → parents[2] resolves to project root
    → project_root / data / raw / <service>
```

## File Selection

Reads the first file found in the service directory.

```python
log_files = [file for file in log_path.iterdir() if file.is_file()]
```

Only files are included — subdirectories are excluded via `is_file()`.

## Return Format

Returns a list of raw strings using `readlines()`. Each string
represents one log line including the newline character at the end.

This format is intentional — the parser receives the list and processes
each line independently.

## Error Handling

Validation happens before any file access — fail fast principle.

```
1. Check if service directory exists → ValueError if not
2. Check if directory contains files → FileNotFoundError if empty
3. Read and return lines
```

Error messages include the service name for clarity:
- `"The service {service} does not exist"`
- `"There are no logs for {service}"`

---

# Design Decisions

## Single file per call
The reader reads one file at a time. This keeps the interface simple
and focused. Reading multiple files or aggregating across services is
a future improvement.

## Raw strings, no processing
The reader returns raw strings. No parsing, no transformation. This
maintains a clean separation of responsibilities between ingestion
and processing layers.

## Pathlib over os.path
`pathlib` provides a cleaner and more readable API for path operations
and is the modern Python standard for file system handling.
